Confluence

In my time managing support and product teams, Atlassian products came up everywhere you turn.  Even though I'd first been interested in Atlassian for their issue tracking took JIRA, I was pleasantly surprised when I discovered Confluence – which you can think of as a more pleasant looking Wiki – it doubles as a very incredible knowledge and collaboration tool that rivals Google Docs and also integrates with many common services (not to mention a whole ecosystem of add-ons) .  When I took to starting my consulting services, I instantly knew that I was going to be using Confluence to help organize and plan my content.

Grammarly

]]>http://localhost:2500/hosting-your-ghost-content-on-github-io/5c6ddded56bda46cb22538b3Wed, 20 Feb 2019 23:17:15 GMTAh, so, I never intended to write this – when I set out to create my portfolio, I knew that I would be eventually hosting it on Github.io, but didn't stop to think how I'd convert my data from Ghost to static HTML files to host on that domain.  After all, Ghost uses a local database to store its content.

I found a helpful blogpost that walked me through the solution, but  after I had to go back and retrace my steps, I thought more attention could be given to the solution.

Note:  I am assuming you already know what Ghost is, and that you already have Ghost installed, and content running developmentally on your localhost.  If not, check out the original post that inspired my post.

It seems there are a few npm utilities out (Buster) that can help convert your ghost content to be statically linked pages, but as pointed out in original blogpost, most systems already have a command installed that can be used. If you simply need a way to get your Ghost content hosted statically, then you just need to consider wget to download your files locally:

wget -r -nH -P dist -E -T 2 -np -k http://localhost:2368/

Running this command will download a static copy of each file linked in the URL and can be parameterized to ensure all your content and inks work the same as hosted at the development version.

I wanna break down this command:

wget

wget a standard utility for fetching web site data from the command line.  If you're using mac or linux, you probably have this installed by default.  If not, it's pretty straightforward to ... uh, get.

-r

We're instructing wget to download files in a recursive manner – meaning wget will first requested the primary document at the given URL (http://localhost:2368/), and then download each documents linked from that document (up to five levels – which can be modified using the -l option).  This makes sure we get all the referenced libaries, style sheets, important children pages linked in the site.

-nH

Disables host-prefixed file names.

-P

Instructs wget to include all files, including images so the desired document looks just like you published

dist

The folder you wish to output the contents to

-E

Adds HTML/CSS extension to the destination file type

-T 2

Retry the request up to 2 times (in case of errors)

-np


Ensures that directories above the parent page are not retrieved

-k


Ensures links are converted to local files


http://localhost:2368/

The default location of your ghost install (run ghost start if not available or check your ghost directory config.development.json for port value)
